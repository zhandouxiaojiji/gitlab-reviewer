# 定时拉取功能说明

## 功能概述

系统已实现后端定时拉取功能，替代了前端的拉取限制。现在系统会自动从GitLab拉取commit和评论数据，并保存到本地JSON文件中。同时支持手动拉取分支信息。

## 主要特性

### 1. 自动定时拉取
- **Commit拉取**: 每5分钟自动拉取一次所有项目的commit数据
- **评论拉取**: 每10秒自动拉取一次commit的评论数据
- **增量拉取**: 只拉取上次拉取时间之后的新数据，提高效率

### 2. 手动拉取功能
- **分支拉取**: 手动拉取项目的分支信息，不进行定时拉取
- **数据刷新**: 可以立即触发指定项目的所有数据拉取
- **按需更新**: 分支信息只在用户需要时才更新

### 3. 本地数据存储
- 数据保存在 `server/data/` 目录下
- 每个项目的commit数据单独存储为 `commits_[projectId].json`
- 每个项目的分支数据单独存储为 `branches_[projectId].json`
- 包含完整的commit信息、评论数据和分支信息

### 4. 手动刷新功能
- 提供手动刷新API接口：`POST /api/gitlab/projects/:projectId/sync`
- 可以立即触发指定项目的commit、评论和分支拉取
- 前端可以添加"刷新"按钮调用此接口

## API变更

### 1. 获取Commit列表
- **接口**: `GET /api/gitlab/projects/:projectId/commits`
- **变更**: 现在从本地JSON文件读取数据，不再实时调用GitLab API
- **优势**: 响应速度快，不受GitLab API限制影响

### 2. 获取分支列表
- **接口**: `GET /api/gitlab/projects/:projectId/branches`
- **变更**: 现在从本地JSON文件读取分支数据，不再实时调用GitLab API
- **优势**: 响应速度快，支持离线查看分支信息
- **提示**: 如果没有分支数据，会提示用户点击刷新按钮

### 3. 手动同步数据
- **接口**: `POST /api/gitlab/projects/:projectId/sync`
- **功能**: 手动触发指定项目的commit、评论和分支拉取
- **用途**: 用户需要立即查看最新数据时使用
- **更新**: 现在会同时拉取分支信息

## 服务器启动流程

1. 服务器启动时自动初始化定时任务
2. 立即执行一次commit和评论拉取
3. 启动定时任务：
   - Commit拉取任务（5分钟间隔）
   - 评论拉取任务（10秒间隔）
4. 分支信息只在手动刷新时拉取
5. 服务器关闭时自动停止所有定时任务

## 性能优化

### 1. 增量拉取策略
- 记录每个项目的最后拉取时间
- 只拉取指定时间之后的新commit
- 避免重复拉取已有数据

### 2. 智能评论拉取
- 只对没有评论或评论可能更新的commit拉取评论
- 已确定有评论的commit不会重复拉取
- 减少不必要的API调用

### 3. 按需分支拉取
- 分支信息不进行定时拉取，减少API调用频率
- 只在用户手动刷新时更新分支信息
- 分支信息变化频率较低，按需拉取更合理

### 4. 错误处理
- 单个项目拉取失败不影响其他项目
- 网络错误时自动重试
- 详细的错误日志记录

## 配置说明

### 拉取间隔配置
可以在 `schedulerService.ts` 中修改拉取间隔：
- `commitPullInterval`: commit拉取间隔（默认5分钟）
- `commentPullInterval`: 评论拉取间隔（默认10秒）
- 分支拉取：仅手动触发，无定时间隔

### 数据存储路径
- 默认存储在 `server/data/` 目录
- 可以通过修改 `DATA_DIR` 常量更改存储路径
- commit数据文件：`commits_[projectId].json`
- 分支数据文件：`branches_[projectId].json`

## 使用建议

1. **首次使用**: 启动服务器后等待几分钟，让系统完成初始数据拉取
2. **数据更新**: 正常情况下commit和评论数据会自动更新，无需手动操作
3. **分支信息**: 首次查看分支列表时，点击刷新按钮拉取分支数据
4. **立即刷新**: 如需查看最新数据，可使用手动同步功能
5. **监控日志**: 查看服务器日志了解拉取状态和可能的错误

## 故障排除

### 常见问题
1. **数据为空**: 检查项目配置是否正确，GitLab访问令牌是否有效
2. **拉取失败**: 查看服务器日志，检查网络连接和GitLab服务状态
3. **数据不更新**: 确认定时任务是否正常运行，检查项目是否设置为活跃状态
4. **分支信息为空**: 点击刷新按钮手动拉取分支信息

### 日志查看
服务器会输出详细的拉取日志，包括：
- 拉取开始和完成时间
- 拉取的commit数量和分支数量
- 错误信息和警告
- 各项目的拉取状态

### 分支相关日志
- 分支拉取开始和完成信息
- 拉取到的分支数量
- 默认分支识别结果
- 分支拉取错误信息
